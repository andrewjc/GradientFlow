Pressure

Gradient Magnitude

Like water pressure in pipes, gradient "pressure" measures how much force is pushing through each layer. Too low means no learning; too high means instability.

Turbulence

Gradient Variance

Unstable, chaotic flow indicates training problems. High turbulence means gradients are fluctuating wildly, making optimization difficult.

Velocity

Gradient Change Rate

How quickly the flow pattern changes over time. High velocity can indicate the network is still adapting; low velocity might mean stagnation.

Blockage

Vanishing Gradients

A clogged pipe where gradients can't flow through. Common in deep networks and RNNs. The layer upstream of the blockage can't learn.

Burst

Exploding Gradients

Over-pressurized pipes about to fail. Gradients grow exponentially, causing NaN values and training collapse. Often occurs in compression layers.

Gradient Scaling

Control mechanisms that regulate flow. The toolkit provides gradient_scale() to act as pressure relief valves at critical points.

Diagnosis Workflow

From symptom to solution in four steps

1

Detect

Run gradient flow analysis on your model. The analyzer hooks into every layer and measures gradient statistics during backpropagation.

2

Diagnose

Review the health report to identify problematic layers. Look for low health scores, high pressure readings, or specific issue types like BOTTLENECK or VANISHING.

3

Fix

Apply targeted fixes based on the diagnosis. Use gradient_scale() for pressure control, add normalization for stability, or restructure bottleneck layers.

4

Verify

Re-run analysis to confirm the fix. Use ComparativeAnalyzer to track improvement across checkpoints and ensure training progresses smoothly.

Common Solutions

Proven fixes for common gradient pathologies

Gradient Scaling - The Pressure Valve 
from gradient_flow import gradient_scale

class ImprovedLayer(nn.Module):
    def forward(self, x):
        # Apply gradient scaling before compression
        x = gradient_scale(x, 0.01)  # Reduce backward pressure

        # Now the compression won't explode gradients
        z = self.compress(x)

        return z
Bottleneck Architecture - Staged Compression 
# BAD: Large single-step compression (256 -> 16)
self.compress = nn.Linear(256, 16)  # 16x gradient amplification!

# GOOD: Two-stage compression
self.compress_down = nn.Linear(256, 64)  # 4x
self.compress_proj = nn.Linear(64, 16)   # 4x
# Total amplification split across layers
Input Normalization - Flow Stabilization 
# Add normalization before problematic layers
self.norm = nn.LayerNorm(input_dim)

def forward(self, x):
    x = self.norm(x)  # Stabilize input magnitude
    x = self.problematic_layer(x)
    return x
